# CEO Plan: SBOM Attestation Wedge + CI Distribution

## Vision

### 10x Check
The 10x version: Sealedge becomes the provenance layer for ALL non-media data, from firmware SBOMs to sensor readings to AI model artifacts. Every connected device ships with Sealedge signing at the source. Every auditor checks Sealedge receipts. The EU CRA makes it mandatory, and Sealedge is the default implementation. That's the trajectory. The SBOM attestation wedge is step one.

### Delight Opportunities Surfaced
1. C2PA compatibility for media profiles (audio, video) -- first amendment auditors, politicians proving what they said
2. Verification badge for READMEs (free distribution)
3. SBOM diff/drift detection across versions
4. Product landing page (accepted)
5. Self-attestation on Sealedge's own releases (accepted)

## Scope Decisions

| # | Proposal | Effort | Decision | Reasoning |

| 1 | C2PA Compatibility Layer | L | DEFERRED | C2PA is media-focused (JUMBF), doesn't fit SBOM use case. Valuable for cam.video/audio profiles later. User noted first amendment auditor use case. |

## Core Deliverables (from design doc)

Design doc: `~/.gstack/projects/TrustEdge-Labs-sealedge/john-main-design-20260401-172433.md`

**Phase 1: SBOM Attestation Wedge**
- New `SbomAttestation` profile in seal-protocols manifest types
- `seal attest-sbom --sbom bom.cdx.json --binary firmware.bin --device-key key.key` CLI command
- CycloneDX JSON support (Phase 1). SPDX deferred to Phase 2.
- Public deployment of platform server (in-memory backend, no auth)
- Static HTML verify page (server-side upload to /v1/verify, render receipt)
- Demo script: 60-second keygen -> attest -> verify flow

**Phase 2: CI Distribution**
- GitHub Action: `sealedge/attest-sbom-action@v1`
- GitLab CI component
- Verification receipts as GitHub Release assets

## Accepted Scope (expansion cherry-picks)
- Product landing page on trustedgelabs-website (what it does, 3-command quick start, public verifier link, GitHub Action link)
- Self-attestation: Sealedge attests its own release builds in CI, .seal archive in every GitHub Release

## Success Metrics
- `seal attest-sbom` works end-to-end: SBOM + binary -> .seal archive -> verified receipt
- Public verification endpoint accessible and responsive
- Demo runs in under 60 seconds
- 3 conversations with engineering leads at companies shipping IoT/connected products into EU, using the demo
