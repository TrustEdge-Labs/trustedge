  <action>
Create `crates/core/build.rs` with the MPL-2.0 copyright header. The script must:

1. Read `PROFILE` env var (set by Cargo to "release" or "debug").
2. Check `CARGO_FEATURE_INSECURE_TLS` env var (set by Cargo when the feature is active; present = enabled).
3. Check `ALLOW_INSECURE_TLS` env var (user-supplied override).
4. If `PROFILE == "release"` AND `CARGO_FEATURE_INSECURE_TLS` is set AND `ALLOW_INSECURE_TLS` is NOT set to "1": call `panic!()` with a clear message.

The panic message must be descriptive. Use this exact text:

```
SECURITY ERROR: The `insecure-tls` feature is enabled in a release build.
This feature disables TLS certificate verification and must never ship in production.

To build release with insecure-tls for testing (NOT for deployment):
    ALLOW_INSECURE_TLS=1 cargo build --release --features insecure-tls

Refusing to compile.
```

Also emit `cargo:rerun-if-env-changed=ALLOW_INSECURE_TLS` so Cargo re-evaluates the guard when the env var changes without a source change.

Do NOT add a `build` key to Cargo.toml — Cargo auto-discovers `build.rs` at the package root.

Full file structure:
```rust
//
// Copyright (c) 2025 TRUSTEDGE LABS LLC
// This source code is subject to the terms of the Mozilla Public License, v. 2.0.
// If a copy of the MPL was not distributed with this file, You can obtain one at https://mozilla.org/MPL/2.0/.
//
// Project: trustedge — Privacy and trust at the edge.
//

fn main() {
    // Re-run if the override env var changes
    println!("cargo:rerun-if-env-changed=ALLOW_INSECURE_TLS");

    let profile = std::env::var("PROFILE").unwrap_or_default();
    let insecure_tls = std::env::var("CARGO_FEATURE_INSECURE_TLS").is_ok();
    let allow_override = std::env::var("ALLOW_INSECURE_TLS").unwrap_or_default() == "1";

    if profile == "release" && insecure_tls && !allow_override {
        panic!(
            "\n\nSECURITY ERROR: The `insecure-tls` feature is enabled in a release build.\n\
             This feature disables TLS certificate verification and must never ship in production.\n\n\
             To build release with insecure-tls for testing (NOT for deployment):\n\
             \n    ALLOW_INSECURE_TLS=1 cargo build --release --features insecure-tls\n\n\
             Refusing to compile.\n"
        );
    }
}
```
