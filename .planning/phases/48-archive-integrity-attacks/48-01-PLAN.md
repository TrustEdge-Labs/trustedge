**SEC-01 Tests (byte mutation):**

`test_sec01_encrypted_chunk_byte_flip()`:
- Call `wrap_encrypted_archive()` to create an encrypted multi-chunk archive
- Read chunk file `chunks/00005.bin` (middle chunk, guaranteed to exist with 64KB/4096 = 16 chunks)
- Flip byte at offset 10 using `wrapping_add(1)`
- Write modified chunk back
- Run `trst verify` with the device pub key
- Assert: exit code 11, stderr contains "hash mismatch" (the BLAKE3 hash of the mutated ciphertext no longer matches the manifest)

`test_sec01_unencrypted_chunk_byte_flip_last_byte()`:
- Call `wrap_unencrypted_archive()`
- Read chunk file `chunks/00000.bin`
- Flip the LAST byte using `wrapping_add(1)` (verifies edge case -- not just first byte)
- Write modified chunk back
- Run `trst verify`
- Assert: exit code 11, stderr contains "hash mismatch"

**SEC-02 Test (chunk injection):**

`test_sec02_injected_extra_chunk()`:
- Call `wrap_unencrypted_archive()`
- Count existing chunk files in chunks/ directory (should be 16)
- Write a new file `chunks/00016.bin` containing 4096 bytes of 0xFF (a spurious chunk not in the manifest)
- Run `trst verify`
- Assert: verification still passes (exit 0) because trst verify only reads chunks listed in the manifest segments array -- the extra file is ignored by the verify logic
- IMPORTANT: This is the actual behavior. The manifest is the source of truth; extra files on disk that are not referenced by the manifest do not cause verification failure. The test documents this behavior.

`test_sec02_injected_chunk_replacing_existing()`:
- Call `wrap_unencrypted_archive()`
- Overwrite `chunks/00005.bin` with 4096 bytes of 0xFF (replacing a real chunk with garbage)
- Run `trst verify`
- Assert: exit code 11, stderr contains "hash mismatch" (replacing a manifest-referenced chunk IS detected)

**SEC-03 Test (chunk reorder):**

`test_sec03_swap_adjacent_chunks()`:
- Call `wrap_unencrypted_archive()`
- Swap contents of `chunks/00003.bin` and `chunks/00004.bin` using a temp file
- Run `trst verify`
- Assert: exit code 11, stderr contains "hash mismatch" (BLAKE3 hashes in manifest no longer match swapped chunk contents)
