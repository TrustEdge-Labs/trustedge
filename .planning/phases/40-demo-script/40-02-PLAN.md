  <action>
Create scripts/demo.sh with the following structure. Use the ci-check.sh step/pass/fail pattern as a style reference but with ANSI colors.

**Header:**
- Copyright header (MPL-2.0, same as ci-check.sh)
- `#!/bin/bash`
- Parse flags: `--local` (force local mode), `--docker` (force docker mode, error if not running)
- `set -euo pipefail`
- Set up ANSI color variables: GREEN='\033[0;32m', RED='\033[0;31m', BLUE='\033[0;34m', BOLD='\033[1m', NC='\033[0m'
- cd to repo root: `cd "$(dirname "$0")/.."`

**Mode detection:**
- TRST command: always use `cargo run -p trustedge-trst-cli --` (works for both modes since demo runs from the repo)
- SERVER_AVAILABLE=false by default
- If not --local: check `curl -sf http://localhost:3001/healthz > /dev/null 2>&1`. If success, SERVER_AVAILABLE=true. If --docker and not available, print error and exit 1.

**Output directory:**
- Create `demo-output/` in CWD (the repo root)
- `DEMO_DIR="demo-output"`
- `rm -rf "$DEMO_DIR" && mkdir -p "$DEMO_DIR"`

**Step functions:**
```bash
STEP=0
TOTAL_STEPS=5  # or 6 if server available (adjusted dynamically)
step_banner() {
    STEP=$((STEP + 1))
    printf "\n${BOLD}${BLUE}[Step %d/%d] %s${NC}\n" "$STEP" "$TOTAL_STEPS" "$1"
}
pass() { printf "  ${GREEN}✔ %s${NC}\n" "$1"; }
fail() { printf "  ${RED}✖ %s${NC}\n" "$1"; FAILURES=$((FAILURES + 1)); }
```

**Steps (6 total when server available, 5 when local-only):**

Compute TOTAL_STEPS: if SERVER_AVAILABLE=true, TOTAL_STEPS=6, else TOTAL_STEPS=5.

Step 1: Generate Device Keys
```bash
step_banner "Generate Ed25519 device key pair"
$TRST keygen --out-key "$DEMO_DIR/device.key" --out-pub "$DEMO_DIR/device.pub"
pass "Created $DEMO_DIR/device.key and $DEMO_DIR/device.pub"
DEVICE_PUB=$(cat "$DEMO_DIR/device.pub" | tr -d '\n')
printf "  Public key: %s\n" "$DEVICE_PUB"
```

Step 2: Generate Sample Data
```bash
step_banner "Generate sample data"
dd if=/dev/urandom of="$DEMO_DIR/sample.bin" bs=1K count=10 2>/dev/null
pass "Created $DEMO_DIR/sample.bin (10 KB random data)"
```

Step 3: Wrap into .trst Archive
```bash
step_banner "Wrap data into .trst archive"
$TRST wrap \
    --profile generic \
    --in "$DEMO_DIR/sample.bin" \
    --out "$DEMO_DIR/sample.trst" \
    --device-key "$DEMO_DIR/device.key" \
    --device-pub "$DEMO_DIR/device.pub" \
    --data-type "sensor" \
    --source "demo-device-01" \
    --description "Demo sensor data capture"
pass "Created $DEMO_DIR/sample.trst archive"
```

Step 4: Local Verification
```bash
step_banner "Verify archive locally"
$TRST verify "$DEMO_DIR/sample.trst" --device-pub "$DEVICE_PUB"
pass "Local verification PASSED"
```

Step 5 (only if SERVER_AVAILABLE): Server Verification
```bash
if [ "$SERVER_AVAILABLE" = true ]; then
    step_banner "Submit to platform verification server"
    $TRST emit-request \
        --archive "$DEMO_DIR/sample.trst" \
        --device-pub "$DEMO_DIR/device.pub" \
        --out "$DEMO_DIR/verify-request.json" \
        --post http://localhost:3001/v1/verify
    pass "Server verification complete — receipt saved"
else
    printf "\n${BOLD}${BLUE}  [Skipped] Server verification (platform not running)${NC}\n"
    printf "  Start the stack: docker compose -f deploy/docker-compose.yml up --build\n"
fi
```

Step N (final): Summary
```bash
step_banner "Summary"
printf "\n  Artifacts in %s/:\n" "$DEMO_DIR"
ls -la "$DEMO_DIR/" | tail -n +2 | while read -r line; do printf "    %s\n" "$line"; done
```

**Final banner:**
```bash
if [ "$FAILURES" -eq 0 ]; then
    printf "\n${BOLD}${GREEN}DEMO COMPLETE — ALL PASSED${NC}\n\n"
    exit 0
else
    printf "\n${BOLD}${RED}DEMO FAILED — %d step(s) failed${NC}\n\n" "$FAILURES"
    exit 1
fi
```

**Error handling:** Each command that could fail should be run with error trapping. If a step fails, call `fail "description"` and continue to next step (do NOT use set -e for the demo steps; use per-command checks). Keep `set -u` and `set -o pipefail`. Wrap each step's commands in `if ... ; then pass ... ; else fail ... ; fi` blocks.

**Important:** Make the script executable with chmod +x.
