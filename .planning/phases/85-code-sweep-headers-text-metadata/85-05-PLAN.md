## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Env-var names (SEALEDGE_*) ↔ runtime configuration | Callers (scripts, CI, local .env files, Docker Compose envs) must update env-var names in lockstep. Stale caller configs would cause fallback-value defaults to be used — a graceful but silent degradation. |
| Clap `#[command(name=...)]` ↔ user-facing --help output + error messages | These attributes are THE string clap prints on the first line of --help and in "error:" messages. Stale attributes would show the wrong binary name to users. |
| Test-vector golden hashes ↔ deterministic byte-content | When test-vector input bytes change (e.g., b"trustedge-test-device" → b"sealedge-test-device"), any downstream BLAKE3 / signing / encryption output that's asserted as a golden value ALSO changes. Failure to regenerate the golden breaks tests — but this is loud failure, not silent. |
| Plaintext test payloads (transport_integration.rs) ↔ round-trip test expectations | The test sends bytes, receives bytes, asserts equality. Changing the brand word in the payload string changes which bytes are sent/received. Internal consistency is maintained within the test itself. No external observer. |
| Inline `//` comments ↔ code-reader understanding | Non-security; documentation-consistency concern. |
