<objective>
Sweep the `.trst` archive extension literal to `.seal` across all executable Rust code — format-string path construction, `ends_with` validation, test fixture paths, test assertion strings, and `anyhow::bail!` invariant messages tied to extension semantics. Doc comments (`///`, `//!`) and CLI help text (`#[command(about = ...)]` brand-word prose) are deferred to Phase 85/86 — only the `.trst` substring inside executable literals renames here.

Purpose: Completes REBRAND-04a by making the on-disk archive extension match the renamed binary (`seal wrap input.bin` produces `input.seal`). Clean break — no reader for legacy `.trst` files.

Output: ~100 `.trst` string literals renamed to `.seal` across 20 .rs files; all affected tests pass under the new extension.
</objective>

**Step 2: Per-file rewrite rules.**

For each file in `<files>`, apply sed to swap `.trst` -> `.seal` in:
- `format!("clip-{}.trst"` -> `format!("clip-{}.seal"`
