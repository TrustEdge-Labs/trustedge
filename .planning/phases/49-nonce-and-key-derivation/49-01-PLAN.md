**SEC-05 tests (nonce uniqueness within an archive):**

`test_sec05_all_chunk_nonces_unique_within_archive`:
- Create a multi-chunk archive using `wrap_unencrypted_archive()` with `--chunk-size 4096` and 64KB input (produces 16 chunks).
- Read each chunk file from `clip.trst/chunks/NNNNN.bin`.
- Extract the first 24 bytes of each chunk file (the nonce).
- Collect all nonces into a `HashSet<[u8; 24]>`.
- Assert that the HashSet size equals the number of chunks (no duplicates).

`test_sec05_nonce_not_all_zeros`:
- Same archive as above.
- Assert that no extracted nonce is all zeros (sanity check that nonces are actually populated).

**SEC-06 tests (nonce uniqueness across archives):**

`test_sec06_same_plaintext_same_key_different_nonces`:
- Create two archives from the same input data using the same device key.
- To use the same device key: run `trst keygen --unencrypted --out key.bin` once, then run `trst wrap` twice with `--device-key key.bin` pointing to the same key file, outputting to two different archive dirs.
- Read chunk 00000.bin from each archive, extract the 24-byte nonce prefix from each.
- Assert the nonces differ (random nonce generation means collision probability is negligible for 24 bytes).

**SEC-07 tests (HKDF key binding):**

`test_sec07_different_device_keys_produce_different_chunk_keys`:
- This is a unit-level test that calls `derive_chunk_key()` directly (it is pub and re-exported from `trustedge_core`).
- Create two different 32-byte inputs (e.g., `[0x01u8; 32]` and `[0x02u8; 32]`).
- Call `derive_chunk_key()` on each.
- Assert the resulting keys are not equal.

`test_sec07_same_device_key_produces_same_chunk_key`:
- Call `derive_chunk_key()` twice with the same input.
- Assert the results are equal (deterministic derivation).

`test_sec07_hkdf_produces_full_entropy_key`:
- Derive a key from a known input.
- Assert the output is not all zeros and not all 0xFF (sanity check).
- Assert the output is exactly 32 bytes.

Follow the SEC-NN_ prefix naming convention from Phase 48 for requirement traceability. Add `#![allow(deprecated)]` for cargo_bin compatibility.
