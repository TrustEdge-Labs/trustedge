Create docs/third-party-attestation-guide.md — a standalone guide for any project to add SBOM attestation (per D-08, D-09).

Structure:

1. **Introduction** — What SBOM attestation is and why (EU CRA, supply chain security). 2-3 sentences.

2. **Prerequisites** — trst binary (link to releases), syft for SBOM generation (link to anchore/syft).

3. **Manual workflow** (copy-paste, per D-09):
   ```bash
   # 1. Generate device keypair
   trst keygen --out-key device.key --out-pub device.pub --unencrypted

   # 2. Generate CycloneDX SBOM
   syft ./my-binary -o cyclonedx-json > sbom.cdx.json

   # 3. Create attestation
   trst attest-sbom --binary ./my-binary --sbom sbom.cdx.json \
     --device-key device.key --device-pub device.pub \
     --out my-binary.te-attestation.json --unencrypted

   # 4. Verify locally
   trst verify-attestation my-binary.te-attestation.json \
     --device-pub "$(cat device.pub)"

   # 5. Upload attestation alongside release binary
   gh release upload v1.0.0 my-binary.te-attestation.json
   ```

4. **CI workflow** (GitHub Actions example, per D-09):
   Show two options:
   a. Using the GitHub Action (one-liner):
      ```yaml
      - uses: TrustEdge-Labs/attest-sbom-action@v1
        with:
          binary: ./target/release/my-app
          sbom: sbom.cdx.json
      ```
   b. Manual CI steps (for non-GitHub CI or more control):
      ```yaml
      - name: Install trst
        run: |
          curl -L https://github.com/TrustEdge-Labs/trustedge/releases/latest/download/trst -o trst
          chmod +x trst
      - name: Attest SBOM
        run: |
          ./trst keygen --out-key key.key --out-pub key.pub --unencrypted
          syft ./my-binary -o cyclonedx-json > sbom.cdx.json
          ./trst attest-sbom --binary ./my-binary --sbom sbom.cdx.json \
            --device-key key.key --device-pub key.pub \
            --out attestation.te-attestation.json --unencrypted
      ```

5. **Verification** — How recipients verify: local CLI verification and public verifier (https://verify.trustedge.dev/verify).

6. **What's in the attestation** — Brief explanation: BLAKE3 hash of binary, BLAKE3 hash of SBOM, full SBOM contents, Ed25519 signature, nonce, timestamp.

Include MPL-2.0 copyright header. Keep it practical — every section has runnable commands.
