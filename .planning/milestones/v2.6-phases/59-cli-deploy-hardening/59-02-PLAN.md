deploy/Dockerfile.dashboard — final stage:

Mechanism (per D-05, Claude's Discretion — use envsubst):
- nginx:alpine includes `envsubst` via gettext package (available by default)
- Rename nginx.conf -> nginx.conf.template; at startup run:
  `envsubst '${SSL_CERT_PATH} ${SSL_KEY_PATH}' < /etc/nginx/conf.d/nginx.conf.template > /etc/nginx/conf.d/default.conf && nginx -g "daemon off;"`
- Template contains the HTTPS server block with `${SSL_CERT_PATH}` and `${SSL_KEY_PATH}` placeholders
- When vars are empty strings, the HTTPS block will have empty paths — guard with nginx `if` or use a separate include approach

Simpler approach: Use `envsubst` and include a `server { listen 443 ssl; ssl_certificate ${SSL_CERT_PATH}; ssl_certificate_key ${SSL_KEY_PATH}; ... }` block, but wrap ssl directives behind a conditional. Since nginx cannot conditionally include server blocks based on env vars at runtime, the cleaner pattern is:
- Keep the HTTPS block in the template
- When SSL_CERT_PATH is empty, nginx will fail to start — so guard the HTTPS block using a separate template file trick:
  - Create `deploy/nginx-ssl.conf.template` — the HTTPS server block
  - In Dockerfile entrypoint: if SSL_CERT_PATH is set, expand and include it; else write empty
  - Use a shell entrypoint script `deploy/docker-entrypoint.sh`

Recommended implementation (clean, idiomatic for nginx:alpine):
1. `deploy/nginx.conf` — keep as HTTP block only (port 80), add this line at top of http context: `include /etc/nginx/conf.d/ssl.conf;` — NO, conf.d/ auto-include will only work if the file exists.

SIMPLEST correct approach:
- `deploy/nginx.conf` — HTTP block on port 80, unchanged
- `deploy/nginx-ssl.conf.template` — HTTPS server block template with `${SSL_CERT_PATH}` and `${SSL_KEY_PATH}`
- `deploy/docker-entrypoint.sh` — shell script that:
  1. If SSL_CERT_PATH and SSL_KEY_PATH are set: `envsubst '${SSL_CERT_PATH} ${SSL_KEY_PATH}' < /etc/nginx/conf.d/nginx-ssl.conf.template > /etc/nginx/conf.d/ssl.conf`
  2. Run `exec nginx -g "daemon off;"`
- Dockerfile.dashboard: COPY both conf files, set ENTRYPOINT to the script

    **Step 1 — deploy/nginx.conf** — leave the HTTP block unchanged. No modifications needed to the HTTP block itself.

    **Step 2 — Create deploy/nginx-ssl.conf.template** with this exact content:
    ```
    server {
        listen 443 ssl;
        server_name _;
        root /usr/share/nginx/html;
        index index.html;

        ssl_certificate     ${SSL_CERT_PATH};
        ssl_certificate_key ${SSL_KEY_PATH};

        ssl_protocols       TLSv1.2 TLSv1.3;
        ssl_ciphers         HIGH:!aNULL:!MD5;

        # SPA fallback
        location / {
            try_files $uri $uri/ /index.html;
        }

        # Cache static assets aggressively
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }

        # Health check endpoint
        location /healthz {
            access_log off;
            return 200 "ok\n";
            add_header Content-Type text/plain;
        }
    }
    ```

    **Step 3 — Create deploy/docker-entrypoint.sh** with this exact content:
    ```bash
    #!/bin/sh
    set -e

    # Conditionally activate HTTPS server block when cert paths are configured.
    # Set SSL_CERT_PATH and SSL_KEY_PATH env vars to enable TLS termination.
    if [ -n "${SSL_CERT_PATH}" ] && [ -n "${SSL_KEY_PATH}" ]; then
        envsubst '${SSL_CERT_PATH} ${SSL_KEY_PATH}' \
            < /etc/nginx/conf.d/nginx-ssl.conf.template \
            > /etc/nginx/conf.d/ssl.conf
    fi

    exec nginx -g "daemon off;"
    ```

    Make it executable: `chmod +x deploy/docker-entrypoint.sh`
  </action>
