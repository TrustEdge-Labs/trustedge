<objective>
Remove VITE_API_KEY from the dashboard client-side bundle entirely. The dashboard only accesses public platform endpoints (/v1/verify, /.well-known/jwks.json, /healthz) — no Bearer token is needed. Pages that called protected endpoints (devices, receipts) show a "not available" message instead. A CI grep guard prevents re-introduction.

Purpose: Eliminate client-side API credential exposure (DASH-01). An API key embedded in a JavaScript bundle is visible to anyone who loads the page.
Output: Dashboard builds with no credential in bundle; CI guard enforces this permanently.
</objective>

**scripts/ci-check.sh** — Add a new step after the existing Step 11 (Secret struct derive check). Insert a "Step 12: Dashboard bundle credential check" block. The check:
1. Runs `npm run build` in `web/dashboard/` only if `node` is available
2. Greps the built `web/dashboard/build/` directory for the string `VITE_API_KEY`
3. Fails if found, passes if not found
4. Skips if `node` is not available or if the `web/dashboard/` directory does not exist

Insert before the `# ── Summary ─────────────────` line:

```bash
# ── Step 12: Dashboard bundle credential check ──────────────────────
step "Step 12: Dashboard bundle credential check"
if [ -d "web/dashboard" ] && command -v node &> /dev/null; then
    if (cd web/dashboard && npm install --silent 2>/dev/null && npm run build --silent 2>/dev/null); then
        if grep -r "VITE_API_KEY" web/dashboard/build/ 2>/dev/null; then
            fail "VITE_API_KEY found in dashboard bundle — credential leak detected"
        else
            pass "No API credentials in dashboard bundle"
        fi
    else
        warn "Dashboard build failed — skipping bundle credential check"
    fi
else
    skip "node not available or web/dashboard not present — skipping bundle check"
fi
```
