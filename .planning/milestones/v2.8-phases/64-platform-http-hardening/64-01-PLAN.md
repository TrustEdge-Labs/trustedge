<objective>
Add proxy-aware IP extraction and Retry-After header to the platform rate limiter.

Purpose: The rate limiter currently uses the direct TCP peer IP, which behind a reverse proxy means all clients share one bucket (the proxy IP). This makes rate limiting ineffective in production (Docker nginx). Additionally, 429 responses lack a Retry-After header, violating RFC 6585 best practice.

Output: Updated rate_limit.rs with trusted proxy support and Retry-After header, updated router.rs to pass config, and integration tests proving both behaviors.
</objective>
