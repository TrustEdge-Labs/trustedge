```rust
// Guard: only 3 bytes of the u32 are used in the nonce (bytes 8..11),

const MAX_CHUNK_INDEX: u64 = 16_777_215; // 2^24 - 1

    ));

```

```rust
/// Maximum chunk index that fits in the 3-byte nonce field (2^24 - 1).
```

**Per D-13, D-14, D-15:**

At the top of main.rs (in the imports section), add a conditional import:
```rust
#[cfg(unix)]
use std::os::unix::fs::PermissionsExt;
```

In `handle_keygen()`, after BOTH `fs::write(&args.out_key, ...)` calls (the unencrypted path at line 342-346 and the encrypted path at line 358-359), add permission enforcement:

```rust
// Set secret key file to owner-only permissions (0600)
#[cfg(unix)]
{
    let perms = std::fs::Permissions::from_mode(0o600);
    std::fs::set_permissions(&args.out_key, perms)
        .with_context(|| format!("Failed to set permissions on {}", args.out_key.display()))?;
}
#[cfg(not(unix))]
{
    eprintln!("Warning: Unable to restrict key file permissions on this platform. Manually restrict access to {}", args.out_key.display());
}
```

Place this AFTER the `fs::write` for the secret key but BEFORE the `fs::write` for the public key (line 362). This way if permission setting fails, we error out before writing the public key.

Do NOT set permissions on the public key file (`args.out_pub`) — per D-13, public key keeps default permissions.

Per D-16, the existing overwrite protection (lines 326-337) is unchanged.
