Modify `deploy/nginx-ssl.conf.template` with three additions:

1. **Security headers (per D-03):** Add the same four security headers from Task 1 at server block level in the existing SSL server block (after `ssl_ciphers` line 11, before `location /`):
   ```nginx
   # Security headers
   add_header X-Content-Type-Options "nosniff" always;
   add_header X-Frame-Options "DENY" always;
   add_header Referrer-Policy "strict-origin-when-cross-origin" always;
   add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; connect-src 'self'; img-src 'self' data:; font-src 'self'" always;
   ```

2. **HSTS header (per D-04):** Add in the SSL server block, alongside the security headers:
   ```nginx
   add_header Strict-Transport-Security "max-age=31536000" always;
   ```
   Conservative: no includeSubDomains, no preload.

3. **HTTP-to-HTTPS redirect server block (per D-05, D-06):** Add a SECOND server block AFTER the existing SSL server block. This block:
   - Listens on port 8080
   - Has a `/healthz` location that returns 200 (per D-06, so docker-compose health probes are not redirected)
   - Has a catch-all that returns 301 to `https://$host:8443$request_uri`
   ```nginx
   # HTTP-to-HTTPS redirect (active only when SSL is enabled)
   server {
       listen 8080;
       server_name _;

       # Health check must not redirect -- docker-compose probes hit port 8080
       location /healthz {
           access_log off;
           return 200 "ok\n";
           add_header Content-Type text/plain;
       }

       location / {
           return 301 https://$host:8443$request_uri;
       }
   }
   ```
   IMPORTANT: The `$host` and `$request_uri` in the redirect are nginx variables (dollar-prefixed), NOT shell variables. They are safe because docker-entrypoint.sh's envsubst only expands `${SSL_CERT_PATH}` and `${SSL_KEY_PATH}` (curly-brace syntax). Nginx `$host` (no curly braces) passes through envsubst untouched.
  </action>
