### Data-Flow Trace (Level 4)

Not applicable — this phase produces a CI workflow (infrastructure configuration), not a component that renders dynamic data. Data flow is traced through the workflow step ordering instead.

**Workflow step ordering:**
1. Build `trst` binary (`cargo build -p trustedge-trst-cli --release`)
2. Generate SBOM from binary path `./target/release/trst` (anchore/sbom-action → `trst-sbom.cdx.json`)
3. Generate ephemeral Ed25519 keypair (`ephemeral.key`, `build.pub`)
4. Bind binary + SBOM into attestation (`trst attest-sbom` → `trst.te-attestation.json`)
5. Upload `trst.te-attestation.json` + `build.pub` to release

Each step produces an output consumed by the next. No step outputs a static/empty value.
