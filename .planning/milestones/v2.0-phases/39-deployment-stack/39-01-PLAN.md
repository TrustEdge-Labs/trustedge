1. Create `deploy/nginx.conf` with the following content:
```nginx
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    # SPA fallback — all routes resolve to index.html for client-side routing
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Cache static assets aggressively
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Health check endpoint for docker-compose
    location /healthz {
        access_log off;
        return 200 "ok\n";
        add_header Content-Type text/plain;
    }
}
```

2. Create `deploy/Dockerfile.dashboard` with the following content:
```dockerfile
# TrustEdge Dashboard — Multi-stage Docker build
#
# Stage 1 (builder): Installs deps and builds SvelteKit static output
# Stage 2 (runtime): Serves static files via nginx:alpine
#
# Build:
#   docker build -f deploy/Dockerfile.dashboard -t trustedge-dashboard web/dashboard
# Run:
#   docker run -p 8080:80 trustedge-dashboard

# ── Stage 1: Builder ──────────────────────────────────────────────────────────

FROM node:20-alpine AS builder

WORKDIR /app

# Accept API base URL as build arg (baked into static JS at build time)
ARG VITE_API_BASE=http://localhost:3001

# Copy package files first for layer caching
COPY package.json package-lock.json* ./

RUN npm install

# Copy source
COPY . .

# Build static output
RUN VITE_API_BASE=${VITE_API_BASE} npm run build

# ── Stage 2: Runtime ──────────────────────────────────────────────────────────

FROM nginx:alpine AS runtime

# Remove default nginx config
RUN rm /etc/nginx/conf.d/default.conf

# Copy custom nginx config (from deploy/ context via docker-compose build context)
COPY nginx.conf /etc/nginx/conf.d/default.conf

# Copy static build output from builder
COPY --from=builder /app/build /usr/share/nginx/html

EXPOSE 80

HEALTHCHECK --interval=10s --timeout=3s --retries=3 \
    CMD wget -qO- http://localhost/healthz || exit 1
```

Note: The Dockerfile.dashboard build context will be `web/dashboard` but nginx.conf needs to be accessible. The docker-compose.yml in plan 02 will handle this by using the repo root as context with the dashboard Dockerfile, OR we place nginx.conf alongside the Dockerfile and reference it. Since docker-compose will set context to repo root, the COPY for nginx.conf in the Dockerfile should reference deploy/nginx.conf relative to the build context.

**Updated approach**: Since docker-compose will use repo root as context (like the platform Dockerfile does), update the Dockerfile.dashboard COPY commands accordingly:
- `COPY web/dashboard/package.json web/dashboard/package-lock.json* ./`
- `COPY web/dashboard/ .`
- `COPY deploy/nginx.conf /etc/nginx/conf.d/default.conf`
