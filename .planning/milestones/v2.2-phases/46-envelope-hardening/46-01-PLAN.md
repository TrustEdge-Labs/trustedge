3. SIMPLIFY `unseal()` (lines 226-291) to remove the v1 fallback. The new unseal() should:
   - Verify the envelope (existing)
   - Sort chunks by sequence (existing)
   - Derive key material once via derive_shared_encryption_key (existing v2 path)
   - Decrypt all chunks via decrypt_chunk_v2 (existing)
   - Zeroize encryption key (existing)
   - Check payload size (existing)
   - Return payload directly -- NO v1 fallback branch

   Remove the entire v1 fallback section (lines 274-290): the `v2_result` variable wrapping, the `if v2_result.is_ok()` early return, and the v1_payload loop. The function should just decrypt and return directly.

   Updated unseal() should look approximately like:
   ```rust
   pub fn unseal(&self, decryption_key: &SigningKey) -> Result<Vec<u8>> {
       if !self.verify() {
           return Err(anyhow::anyhow!("Envelope verification failed"));
       }
       let mut sorted_chunks = self.chunks.clone();
       sorted_chunks.sort_by_key(|chunk| chunk.sequence);
       let total_chunks = sorted_chunks.len();
       let sender_public_key = VerifyingKey::from_bytes(&self.verifying_key_bytes)
           .context("Invalid sender public key in envelope")?;
       let (mut encryption_key, nonce_prefix) =
           derive_shared_encryption_key(decryption_key, &sender_public_key, &self.hkdf_salt)?;
       let mut payload = Vec::new();
       for (i, chunk) in sorted_chunks.iter().enumerate() {
           let is_last = i == total_chunks - 1;
           let decrypted = self.decrypt_chunk_v2(chunk, &encryption_key, &nonce_prefix, is_last)?;
           payload.extend_from_slice(&decrypted);
       }
       encryption_key.zeroize();
       if payload.len() != self.metadata.payload_size as usize {
           return Err(anyhow::anyhow!(
               "Payload size mismatch: expected {}, got {}",
               self.metadata.payload_size,
               payload.len()
           ));
       }
       Ok(payload)
   }
   ```
