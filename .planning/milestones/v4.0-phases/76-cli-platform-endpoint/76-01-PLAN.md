**6. Implement handle_attest_sbom** (per D-01 through D-06, D-16):
- Validate inputs FIRST (per D-04, CLI-03):
  - Check binary file exists, then check metadata: if file size is 0, print "Error: binary file is empty (0 bytes)" to stderr and return CliExitError with code 1.
  - Check binary file size <= 256MB (256 * 1024 * 1024 bytes). If over, print "Error: binary file exceeds 256 MB limit ({size} bytes)" and return CliExitError code 1.
  - Check SBOM file exists, read contents, attempt `serde_json::from_str::<serde_json::Value>()`. If it fails, print "Error: SBOM file is not valid JSON" and return CliExitError code 1. (per D-05)
- If `args.unencrypted`, call `warn_unencrypted()`.
- Load keypair using same pattern as handle_wrap: read device_key file, check `is_encrypted_key_file()`, if encrypted prompt passphrase via `rpassword::prompt_password("Enter passphrase for device key: ")` then `DeviceKeypair::from_encrypted_file()`, else `DeviceKeypair::from_unencrypted_file()`. If `--unencrypted` flag set, skip encryption check and use `from_unencrypted_file()` directly.
- Read device_pub file content and trim it.
- Call `PointAttestation::create(&args.binary, "binary", &args.sbom, "sbom", &keypair)`.
- Serialize with `attestation.to_json()`.
- Determine output path: `args.out.unwrap_or_else(|| PathBuf::from("attestation.te-attestation.json"))` (per D-02).
- Write JSON to output file. Set permissions to 0o644 on Unix (per D-06).
- Print success: `"✔ Attestation written to {path}"` to stderr, and print the public key and subject/evidence hashes.

**7. Implement handle_verify_attestation** (per D-07 through D-10, D-16):
- Read attestation file, call `PointAttestation::from_json()`.
- Resolve device_pub: if the string starts with "ed25519:" use it directly, otherwise treat it as a file path, read the file, and trim.
- Call `attestation.verify_signature(&device_pub)`.
- If --binary or --sbom provided, call `attestation.verify_file_hashes(binary_path, sbom_path)` (per D-08). On HashMismatch, return CliExitError code 10.
- Print human-readable output to stdout (per D-10): format string, public key, timestamp, subject hash + filename, evidence hash + filename, and "Signature: VERIFIED" or "Signature: FAILED".
- Exit codes (per D-09, D-16): 0 = verified, 1 = IO/JSON error (propagated by anyhow), 10 = signature failed or hash mismatch (via CliExitError).
