  <action>
Create `scripts/demo-attestation.sh` following the same patterns as `scripts/demo.sh` (per D-12: separate script, does NOT modify demo.sh).

**Script structure (mirror demo.sh patterns):**
- Copyright header (MPL-2.0)
- `set -uo pipefail`, `cd "$(dirname "$0")/.."`
- ANSI color constants (GREEN, RED, BLUE, BOLD, NC)
- Argument parsing: `--local` (skip remote), `--endpoint <url>` (override endpoint URL, default https://verify.trustedge.dev or whatever the public URL is)
- Step counter, pass/fail helpers (same as demo.sh)
- Timing: capture START_TIME at beginning, compute elapsed at end

**Flow (per D-13):**

Step 1: Check prerequisites
- Check `cargo` is available (needed to build trst)
- Check `syft` is available. If not: print install instructions ("Install syft: curl -sSfL https://raw.githubusercontent.com/anchore/syft/main/install.sh | sh -s") and exit 1. Per Claude's discretion: do NOT auto-install, just provide instructions.
- Set TRST="cargo run -q -p trustedge-trst-cli --"

Step 2: Generate Ed25519 device key pair
- `$TRST keygen --out-key "$DEMO_DIR/device.key" --out-pub "$DEMO_DIR/device.pub" --unencrypted`
- Read DEVICE_PUB from device.pub

Step 3: Build the trst binary (for self-attestation)
- `cargo build -p trustedge-trst-cli --release 2>&1`
- Set TRST_BINARY=target/release/trst

Step 4: Generate SBOM with syft
- `syft "$TRST_BINARY" -o cyclonedx-json > "$DEMO_DIR/sbom.cdx.json" 2>/dev/null`
- This is the ~10-20s step per D-14

Step 5: Create attestation
- `$TRST attest-sbom --binary "$TRST_BINARY" --sbom "$DEMO_DIR/sbom.cdx.json" --device-key "$DEMO_DIR/device.key" --device-pub "$DEMO_DIR/device.pub" --out "$DEMO_DIR/attestation.te-attestation.json" --unencrypted`
- pass/fail output

Step 6: Verify attestation locally
- `$TRST verify-attestation "$DEMO_DIR/attestation.te-attestation.json" --device-pub "$DEVICE_PUB"`
- pass/fail output

Step 7: Remote verification (optional, per D-15)
- Auto-detect: `curl -sf "$ENDPOINT/healthz" > /dev/null 2>&1`
- If reachable: `curl -sf -X POST "$ENDPOINT/v1/verify-attestation" -d @"$DEMO_DIR/attestation.te-attestation.json"` and show result
- If not reachable: print "[Skipped] Remote verification (endpoint not reachable at $ENDPOINT)" and note it
- Skip entirely if `--local` flag

Step 8: Summary
- List artifacts in $DEMO_DIR
- Print elapsed time
- Print total pass/fail

**Output formatting (per D-16):**
- Step numbers: `[Step N/M] Description`
- ✔ for pass, ✖ for fail, ⚠ for warning
- Timing: "Completed in Xs" at the end
- Header: "● TrustEdge SBOM Attestation Demo"

**Demo directory:** `demo-attestation-output/` (separate from demo.sh's `demo-output/`)
- `rm -rf "$DEMO_DIR" && mkdir -p "$DEMO_DIR"` at start

**Make executable:** `chmod +x scripts/demo-attestation.sh`

**Do NOT:**
- Modify scripts/demo.sh (per D-12)
- Auto-install syft (Claude's discretion: error with instructions is safer)
- Require Docker or platform server for the basic flow
- Use `cargo run` for syft step (syft is a standalone binary)
