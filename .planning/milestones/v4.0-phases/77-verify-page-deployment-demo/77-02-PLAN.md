**deploy/digitalocean/Dockerfile:**
Based on the existing `deploy/Dockerfile` pattern but for verify-only mode:
- Stage 1 (builder): `rust:1.88-slim`, install pkg-config + libssl-dev, copy workspace, build with `cargo build -p trustedge-platform-server --features http --release` (NO postgres feature, per D-07)
- Stage 2 (runtime): `debian:bookworm-slim`, install ca-certificates + libssl3 + wget, create non-root `trustedge` user, copy binary, EXPOSE 3001
- CMD: `["trustedge-platform-server", "serve"]`
- Add copyright header as comment at top
- Add HEALTHCHECK instruction: `HEALTHCHECK --interval=30s --timeout=5s CMD wget -qO- http://localhost:3001/healthz || exit 1`

**deploy/digitalocean/app.yaml:**
DigitalOcean App Platform spec (doctl format):
```yaml
name: trustedge-verifier
services:
  - name: platform
    dockerfile_path: deploy/digitalocean/Dockerfile
    source_dir: /
    github:
      repo: TrustEdge-Labs/trustedge
      branch: main
      deploy_on_push: true
    instance_size_slug: basic-xxs
    instance_count: 1
    http_port: 3001
    health_check:
      http_path: /healthz
      initial_delay_seconds: 10
      period_seconds: 30
    envs:
      - key: PORT
        value: "3001"
      - key: RECEIPT_TTL_SECS
        value: "3600"
      - key: RATE_LIMIT_RPS
        value: "10"
      - key: RUST_LOG
        value: "info"
```
- Region: leave unset (DO defaults to nearest)
- Instance size: basic-xxs (smallest, cheapest — demo stage per D-08)
- No database, no workers, no static sites (all-in-one binary serves HTML per Plan 01)

**deploy/digitalocean/.env.example:**
Document the minimal env vars for verify-only deployment:
- PORT (default 3001)
- RECEIPT_TTL_SECS (default 3600)
- RATE_LIMIT_RPS (default 10)
- RUST_LOG (default info)
- Note: No DATABASE_URL needed (verify-only mode)

**Do NOT:**
- Include postgres feature or DATABASE_URL
- Add nginx sidecar or separate static site (HTML is embedded in binary)
- Add custom domain config (deferred)
- Add TLS config (DO App Platform handles TLS termination)
