1. Change `build()` (line 153) to panic when the placeholder is used outside tests:

```rust
pub fn build(self) -> CAConfig {
    if self.jwt_secret == "your-secret-key" && !cfg!(test) {
        panic!(
            "CAConfig: placeholder JWT secret 'your-secret-key' must not be used outside tests. \
             Set a real secret via CAConfigBuilder::jwt_secret()."
        );
    }
    CAConfig {
        database_url: self.database_url,
        jwt_secret: Secret::new(self.jwt_secret),
        ca_name: self.ca_name,
        ca_organization: self.ca_organization,
        ca_country: self.ca_country,
        certificate_validity_days: self.certificate_validity_days,
    }
}
```

Using `cfg!(test)` allows tests to still use the default placeholder (it compiles to `true` in test builds, `false` in production). Per D-08, `CAConfig::default()` and `CAConfigBuilder::default()` keep their current values.

2. Update existing tests (per D-07) to use an explicit test secret where they call `.build()` without setting a jwt_secret. Update these tests:

- `test_caconfig_debug_redacts_jwt_secret` (line 170): Change `CAConfig::default()` to `CAConfig::builder().jwt_secret("test-jwt-secret-do-not-use-in-prod".to_string()).build()`. Update the assertion to check it does NOT contain `test-jwt-secret-do-not-use-in-prod` instead of `your-secret-key`.

- `test_caconfig_builder_defaults` (line 196): This test verifies the default jwt_secret is `"your-secret-key"`. Since we allow placeholder in test mode (`cfg!(test)` is true), this test can remain as-is. But update it to explicitly document this: add a comment `// In test builds, cfg!(test) allows the placeholder through build()`.
