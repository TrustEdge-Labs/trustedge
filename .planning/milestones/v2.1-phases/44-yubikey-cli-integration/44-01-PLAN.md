    - Test: verify_manifest with "ecdsa-p256:<base64>" public key and wrong data returns Ok(false)
    - Test: verify_manifest with "ed25519:<base64>" still works (regression)
    - Test: verify_manifest with unknown prefix returns Err(CryptoError::InvalidSignatureFormat)
  </behavior>
  <action>
    1. Add `use p256::ecdsa::{signature::Verifier as _, Signature as P256Signature, VerifyingKey as P256VerifyingKey};` to imports in crypto.rs.

    2. Add a new function `verify_manifest_ecdsa_p256(device_public: &str, canonical_bytes: &[u8], signature_str: &str) -> Result<bool, CryptoError>`:
       - Strip "ecdsa-p256:" prefix from `device_public`, base64-decode to get the SEC1 compressed/uncompressed public key bytes.
       - Construct `P256VerifyingKey::from_sec1_bytes(&pub_bytes)`.
       - Strip "ecdsa-p256:" prefix from `signature_str`, base64-decode to get DER-encoded ECDSA signature bytes.
       - Construct `P256Signature::from_der(&sig_bytes)`.
       - IMPORTANT: The YubiKey signs SHA-256 pre-hashed digests. The `p256::ecdsa::VerifyingKey::verify()` method hashes the message itself (it does SHA-256 internally). So pass `canonical_bytes` directly to `verifying_key.verify(canonical_bytes, &signature)` -- the p256 crate handles the hashing.
       - Return Ok(true) on success, Ok(false) on signature mismatch.

    3. Modify existing `verify_manifest()` to dispatch on signature prefix:
       - If `signature_str` starts with "ed25519:" -> existing Ed25519 path (also require `device_public` starts with "ed25519:").
       - If `signature_str` starts with "ecdsa-p256:" -> call `verify_manifest_ecdsa_p256()` (also require `device_public` starts with "ecdsa-p256:").
       - Otherwise -> return `Err(CryptoError::InvalidSignatureFormat("Unsupported signature algorithm. Expected ed25519: or ecdsa-p256: prefix".to_string()))`.

    4. Add unit tests in the existing `mod tests` block:
       - `test_ecdsa_p256_sign_verify_round_trip`: Generate a P-256 signing key with `p256::ecdsa::SigningKey::random(&mut OsRng)`, sign canonical bytes using `p256::ecdsa::signature::Signer::sign()`, format as "ecdsa-p256:<base64_der>", verify via `verify_manifest()`.
       - `test_ecdsa_p256_wrong_data`: Same setup, verify with different data returns Ok(false).
       - `test_ed25519_still_works`: Existing `test_round_trip_sign_verify` covers this but add explicit assertion comment.
       - `test_unknown_signature_prefix`: Call verify_manifest with "rsa:" prefix, expect Err.
  </action>
