From crates/core/src/backends/yubikey.rs:
- `piv_sign()` returns DER-encoded ECDSA signature bytes
- `piv_get_public_key()` returns DER-encoded SubjectPublicKeyInfo
- YubiKey does SHA-256 pre-hash internally before ECDSA signing
- p256 crate's `Verifier::verify()` also hashes internally
- CRITICAL: This means the YubiKey signs SHA-256(data), and p256's verify expects to hash the data too. The piv_sign function pre-hashes with SHA-256 and passes the digest. The p256 verify function hashes the original data. These are compatible: piv_sign(SHA256(canonical_bytes)) produces a signature that p256::verify(canonical_bytes) will accept, because piv_sign signs the digest directly while p256::verify computes SHA256(canonical_bytes) and verifies against that same digest.
