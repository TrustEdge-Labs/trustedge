Use GNU sed in-place with backup, cross-platform-safe:

```bash
# Rust files only — main workspace + experimental + examples
while IFS= read -r f; do
  sed -i \
    -e 's|// Project: trustedge — Privacy and trust at the edge\.|// Project: sealedge — Privacy and trust at the edge.|g' \
    -e 's|// GitHub: https://github\.com/TrustEdge-Labs/trustedge|// GitHub: https://github.com/TrustEdge-Labs/sealedge|g' \
    "$f"

while IFS= read -r f; do
  sed -i \
    -e 's|// Project: trustedge — Privacy and trust at the edge\.|// Project: sealedge — Privacy and trust at the edge.|g' \
    -e 's|// GitHub: https://github\.com/TrustEdge-Labs/trustedge|// GitHub: https://github.com/TrustEdge-Labs/sealedge|g' \
    "$f"
done < /tmp/85-04-experimental-files.txt
```

**IMPORTANT:** The em-dash `—` in `Project: trustedge — Privacy` is a real em-dash character (U+2014), not a hyphen. Sed handles UTF-8 transparently but the shell/terminal must be UTF-8 locale. Set `LC_ALL=C.UTF-8` (or system equivalent) before running if uncertain. Alternatively, use perl:

```bash
# Perl variant — UTF-8 native, same result
while IFS= read -r f; do
  perl -CSAD -i -pe 's|// Project: trustedge(?= — Privacy)|// Project: sealedge|g; s|// GitHub: https://github\.com/TrustEdge-Labs/trustedge|// GitHub: https://github.com/TrustEdge-Labs/sealedge|g' "$f"
done < /tmp/85-04-files.txt
# (same for experimental)
```

Planner's choice: use sed if UTF-8 locale confirmed; otherwise perl.
