<objective>
Fix the double-decrypt bug in trst-wasm that corrupts browser-based archive chunk decryption, wire the crypto module into the crate build, add missing dependencies, and prove correctness with a round-trip test.

Purpose: The trst-wasm crate's decrypt() function chains .decrypt() twice on the cipher object, producing garbage output (or a panic) instead of plaintext. The module was never wired into lib.rs so the bug went undetected. This is the P0 fix for browser-based archive verification.

- crates/trst-wasm/src/lib.rs with `pub mod crypto;` wiring
